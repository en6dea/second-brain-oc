# Second Brain OS V86.12 — Finance, PWA и синхронизация

## Что исправлено

### Планирование по месяцам
- Повторяющийся расход хранится отдельной записью в каждом месяце.
- Расход 6 000 ₽ на 4 месяца отображается как 6 000 ₽ в каждом выбранном месяце, а не 24 000 ₽ в одном.
- У серии отображается номер платежа: 1 из 4, 2 из 4 и так далее.
- На главной странице финансов показывается сумма ближайшего конкретного месяца.

### Скорость финансов
- Расчёты операций, категорий, обязательности и лимитов выполняются одним проходом.
- Добавлен кэш финансового периода, который сбрасывается только после изменения данных.
- При 2 000 тестовых операциях экран финансов открылся примерно за 110 мс в автоматическом smoke-тесте.
- Во время перехода показывается короткий индикатор, поэтому интерфейс не выглядит зависшим.

### Категории
- После изменения обязательности, лимита, названия или иконки пользователь возвращается к той же строке и позиции списка.
- Архивные категории исключены из активных выпадающих списков.
- Архивные категории можно восстановить.
- Категориям без нормальной иконки автоматически назначаются подходящие живые иконки.
- Менеджер категорий адаптирован под компьютер, планшет и телефон без горизонтального наложения.

### Долги
- В форму долга добавлены сумма ближайшего платежа и график: разовый или ежемесячный.
- Активные долги с датой автоматически появляются в планировании нужного месяца.
- Ежемесячный платёж появляется в каждом месяце начиная с указанной даты.
- Автоматический платёж не создаёт операцию сам и не дублирует данные: он остаётся прогнозом до фактической оплаты.

## Установка как приложение

### iPhone
1. Откройте сайт в Safari.
2. Нажмите «Поделиться».
3. Выберите «На экран Домой».
4. Запускайте Second Brain OS с иконки как отдельное приложение.

### Windows / Mac
1. Откройте сайт в Chrome или Edge.
2. Нажмите значок установки в адресной строке или кнопку «Установить приложение» в настройках.
3. Приложение откроется в отдельном окне и будет доступно из меню «Пуск» / Applications.

## Синхронизация iPhone и компьютера

Статический GitHub Pages не имеет собственной базы данных. В V86.12 подготовлена синхронизация через Firebase Authentication + Firestore.

1. Создайте проект в Firebase Console.
2. Включите Authentication → Email/Password.
3. Создайте Firestore Database.
4. Примените файл `firestore.rules` из сборки.
5. Добавьте Web App и скопируйте настройки.
6. Вставьте настройки в `firebase-config.js` вместо значений `PASTE_...`.
7. Загрузите файлы в GitHub.
8. На iPhone и компьютере откройте «Настройки», войдите под одним email.

После входа:
- изменения автоматически отправляются в облако;
- другое устройство получает более свежую версию;
- локальное сохранение и IndexedDB продолжают работать;
- при отсутствии интернета изменения остаются локально и отправляются после восстановления связи.
