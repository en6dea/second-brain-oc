# Second Brain OS V95 — Finance Active

## Исправленная проблема

После V94 раздел «Финансы» мог открываться в аварийном режиме «Финансы открыты». Карточки аварийной страницы выглядели как кнопки, но основной финансовый экран не загружался. В техническом сообщении отображалось:

`tips.unshift is not a function`

Причина: один слой финансового помощника возвращал объект `{ data, tips }`, а последующая обёртка в части сценариев воспринимала результат как массив и пыталась вызвать `unshift` не у того значения.

## Что изменено

- Нормализован контракт `financeAssistant()`.
- Независимо от старого формата результат приводится к объекту `{ data, tips }`.
- Новые предупреждения добавляются только в `assistant.tips`.
- Восстановлен обычный экран «Финансы» вместо аварийной заглушки.
- Кнопка/вкладка «Операции» снова открывает полноценный список операций.
- Импорт банковского CSV из V94 сохранён и проверен: кнопка открывает окно выбора CSV-файла.
- Обновлены V95 build-метки, manifest, Service Worker и PWA cache-busting.
- Основной ключ данных остаётся `secondBrainOS.v1`.
- Очистка `localStorage`, IndexedDB и пользовательских данных не добавлялась.
- Перед первым применением V95 создаётся отдельная резервная копия с ключами V95.

## Изменённые рабочие файлы

- `index.html`
- `app-v93-assistant.js`
- `manifest.webmanifest`
- `pwa-v88.js`
- `sw.js`

## Проверенный сценарий

1. Открыть «Финансы».
2. Убедиться, что показан обычный финансовый экран, а не «Безопасный режим финансов».
3. Нажать «Операции».
4. Убедиться, что открылась страница операций.
5. Нажать «Импорт CSV из банка».
6. Убедиться, что открылось модальное окно с выбором `.csv`-файла.
