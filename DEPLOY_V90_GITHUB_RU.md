# Публикация Second Brain OS V90 в GitHub Pages

## Вариант 1 — загрузить полный архив

1. Скачайте `second-brain-oc-v90-github-ready.zip`.
2. Распакуйте архив на компьютере.
3. Откройте репозиторий `en6dea/second-brain-oc` на GitHub.
4. Загрузите содержимое распакованной папки в корень репозитория с заменой одноимённых файлов.
5. Подтвердите commit.
6. Дождитесь завершения GitHub Pages / Actions.

## Вариант 2 — загрузить только обновление

Загрузите содержимое `second-brain-oc-v90-upload-files.zip` в корень текущего репозитория. Архив содержит изменённые и новые файлы V90.

## Первый запуск

Откройте:

`https://en6dea.github.io/second-brain-oc/?v=90#today`

Проверьте метку в левом нижнем углу:

`V90 · VERIFIED STABILITY`

## Если всё ещё открывается старая версия

1. Закройте установленное PWA-приложение Second Brain OS, если оно открыто.
2. Откройте ссылку с `?v=90` в обычной вкладке браузера.
3. Выполните жёсткое обновление страницы: `Ctrl + F5`.
4. Подождите 5–10 секунд и обновите ещё раз — новый Service Worker должен активироваться и удалить старые кэши интерфейса.

**Не используйте очистку всех данных сайта в браузере**, если предварительно не экспортировали данные: полная очистка site data может удалить `localStorage` и IndexedDB. V90 сам обновляет только кэш интерфейса.

## Быстрая проверка после публикации

- Перейдите по всем пунктам левого меню.
- Откройте «Финансы» → «Настроить расчёт» и введите тестовое число, затем сохраните.
- Создайте или откройте событие календаря, измените продолжительность и сохраните.
- Перетащите событие на другой день.
- Удалите тестовое событие и нажмите Undo.
- Откройте редактирование привычки, смените иконку, сохраните и перезагрузите страницу.
- Проверьте, что в очереди «Разобрать» нет слова `transfer` и наложения текста.

## Откат

Перед публикацией скачайте текущую версию репозитория. Для отката верните предыдущие версии 6 основных файлов:

- `index.html`
- `manifest.webmanifest`
- `pwa-v88.js`
- `sw.js`
- удалите `app-v90-stability.js`
- удалите `styles-v90-stability.css`

Пользовательские данные при обычном откате файлов не должны затрагиваться.
