# Second Brain OS V89 — критическая стабилизация интерфейса

## Что изменено

Релиз выполнен поверх текущей архитектуры без очистки `localStorage`, `IndexedDB` и без замены пользовательских данных демоданными.

1. **Модальные окна**
   - принудительно перемещаются в корень `body`;
   - открываются фиксированным слоем поверх приложения;
   - блокируют прокрутку фона;
   - закрываются по `Escape`, клику по затемнению и штатной кнопке;
   - добавлены возврат фокуса и удержание клавиатурного фокуса внутри окна.

2. **Календарь**
   - сохранены создание и редактирование через текущую форму;
   - добавлены видимые действия: на день раньше, на день позже, удалить;
   - добавлено перетаскивание карточек между днями недели и месяца;
   - при переносе изменяется существующая запись, новая не создаётся;
   - удаление требует подтверждения и использует архивную копию, когда доступен `archiveSnapshot`.

3. **Кликабельность**
   - восстановлены корректные `pointer-events` и минимальная область нажатия;
   - визуальные кнопки, карточки и действия не перекрываются декоративными слоями;
   - добавлены заметные состояния клавиатурного фокуса.

4. **Фирменные иконки**
   - иконки навигации заменяются идемпотентно;
   - каждому разделу назначен стабильный фирменный цвет;
   - активный раздел использует цветовую инверсию: белая иконка на цветном градиенте;
   - фиксированная геометрия иконок исключает скачок при переходах.

5. **Верстка**
   - карточки очереди автоматически растут по высоте;
   - длинные названия и описания переносятся;
   - действия календаря больше не зажимают текст;
   - мобильные модалки открываются как нижний лист без ухода вниз страницы.

6. **Сохранность данных**
   - при первом запуске V89 создаётся сырая localStorage-копия;
   - создаётся IndexedDB-снимок `backup:v89-before-critical:*`;
   - миграция не меняет ключи коллекций и не удаляет записи.

## Развёртывание

Загрузить содержимое архива в корень репозитория `en6dea/second-brain-oc`, заменив одноимённые файлы. После GitHub Pages открыть приложение с параметром `?v=89`.

Пользовательские данные при обновлении не очищать. `force-update.html` применять только для очистки Service Worker/Cache Storage, не хранилища данных.

## Проверка релиза

- JavaScript проверен командой `node --check`.
- Manifest проверен как валидный JSON.
- В браузерном QA-прогоне проверены: кликабельность навигации, фиксированное модальное окно, создание/редактирование/перенос/удаление календарной записи, архивная копия, отметка привычки, очередь без горизонтального переполнения, открытие финансов и мобильная недопустимость общего горизонтального скролла.
- В каталоге `qa-v89` сохранены отчёт и контрольные скриншоты.

## Известное ограничение QA-среды

Полноценный тест перезагрузки с реальными `localStorage`/`IndexedDB` в автоматизированной песочнице не выполнялся: браузерная политика блокирует навигацию на локальный origin, а `about:blank` запрещает доступ к хранилищам. Код резервного копирования построен поверх уже используемых приложением ключей и не содержит операций очистки. Перед выкладкой всё равно рекомендуется скачать штатный экспорт данных из текущей версии.
