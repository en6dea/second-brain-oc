# Следующий архитектурный этап

V85 остаётся автономным приложением GitHub Pages и хранит данные локально. Для полноценной бесперебойной работы рекомендуется отдельная миграция без изменения дизайна:

1. Разделить интерфейс на модули и компоненты.
2. Добавить облачную PostgreSQL-базу, авторизацию и хранилище изображений.
3. Сохранить IndexedDB как локальный offline-first слой.
4. Добавить очередь синхронизации и разрешение конфликтов.
5. Ввести мягкое удаление, версии записей и историю изменений.
6. Настроить автоматические резервные копии.
7. Запускать тесты и деплой через GitHub Actions.
8. Добавить мониторинг ошибок и экран состояния синхронизации.

Переход должен выполняться отдельным релизом после резервного копирования и тестовой миграции копии данных.
