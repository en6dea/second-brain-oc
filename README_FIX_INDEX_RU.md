# V86.1 — восстановление приложения после цикла обновления

## Причина
В репозитории файл `index.html` был заменён маленькой страницей обновления размером около 2,21 КБ. Поэтому главная ссылка снова и снова открывала экран «Обновляем интерфейс». Это не проблема кэша: сама главная страница репозитория стала страницей обновления.

## Что загрузить
Загрузите в корень репозитория с заменой:

- `index.html` — настоящее приложение, размер около 419 КБ;
- `app-v86-direct-20260721.html` — резервный прямой вход в приложение;
- `force-update.html` — безопасная страница очистки без автоперехода.

## Проверка перед открытием
На GitHub откройте `index.html`. Его размер должен быть около 419 КБ, а не 2,21 КБ.

## После публикации
Сначала откройте прямую страницу:

`https://en6dea.github.io/second-brain-oc/app-v86-direct-20260721.html?v=861#today`

Она использует тот же origin, поэтому существующие localStorage и IndexedDB остаются доступными.

После проверки можно открыть обычную главную ссылку:

`https://en6dea.github.io/second-brain-oc/?v=861#today`

`force-update.html` больше не перенаправляет страницу автоматически.
