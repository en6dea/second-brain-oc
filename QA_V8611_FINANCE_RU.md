# QA V86.11 — Финансы

Проверено в Chromium:

- открывается главная страница финансов;
- открывается планирование будущего месяца;
- сохраняются ожидаемый доход, день зарплаты и резерв;
- автоматическое распределение создаёт лимиты категорий;
- карточки обязательных и необязательных расходов открывают отфильтрованные операции;
- карточка общего лимита прокручивает к таблице лимитов;
- лимит категории сохраняется прямо из строки аналитики;
- в форме операции доступен третий тип «Перевод / нейтральное движение»;
- обязательность скрывается для доходов и переводов;
- нейтральные операции исключаются из суммы доходов, расходов и чистого потока;
- список категорий сохраняет позицию прокрутки после настройки;
- удалённая категория переносится в архив, исчезает из активных списков и восстанавливается;
- окно категорий содержит отдельную группу нейтральных движений;
- JavaScript прошёл `node --check`.

Тестовые ошибки хранилища при запуске через `page.set_content` связаны с изолированным браузерным контекстом QA и не относятся к GitHub Pages.
