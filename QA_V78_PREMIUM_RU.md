# QA V78 Premium Unified

## Проверено автоматически

- JavaScript `app-v78-premium.js` проходит `node --check`.
- Все новые пользовательские маршруты формируют HTML без исключений:
  - Сегодня
  - Привычки
  - отдельная страница привычки
  - Информация
  - Финансы
  - Операции
  - Анализ категорий
  - Экспорт CSV
  - Обязательства
  - Профиль
  - Вишлист привычек
- Проверено формирование страниц на пустом состоянии и на тестовом состоянии с операциями, долгами, покупками, людьми и заметками.
- Все ресурсы из `CORE_ASSETS` service worker присутствуют в проекте.
- V77 JS/CSS не подключаются в `index.html` и не входят в новый cache manifest.
- В новом модуле нет вызовов очистки `localStorage`, удаления IndexedDB или сброса основных коллекций.
- CSV строится из полного массива операций; пагинация интерфейса не ограничивает экспорт.
- Профиль хранится в совместимом объекте `state.settings.profile`.
- Проверено открытие прежних разделов внутри новой V78-оболочки: Люди, Заметки, Состояние Полины, Пароли, Архив, Календарь и Настройки.

## Защита данных

Миграция V78 только добавляет недостающие поля и не очищает существующие записи. Перед обновлением всё равно необходимо скачать JSON-экспорт из текущей версии.

## Ограничение проверки

В контейнере не удалось стабильно запустить установленный Chromium для полноценного визуального end-to-end теста. Поэтому выполнены синтаксические, статические и маршрутные smoke-тесты, но финальную проверку кликов в опубликованном GitHub Pages нужно провести после загрузки файлов.
