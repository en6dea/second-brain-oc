# Безопасная публикация Second Brain OS

## Главное правило

В публичный репозиторий загружается только код приложения. Личные резервные копии, `data.json`, банковские CSV, базы данных и реальный `firebase-config.js` не загружаются.

## Рекомендуемая схема

1. Старый репозиторий `en6dea/second-brain-oc` оставить приватным до завершения переноса.
2. Создать новый публичный репозиторий без начального README, например `second-brain-life`.
3. Распаковать чистый релизный ZIP в отдельную новую папку.
4. Проверить, что в папке нет `data.json`, `firebase-config.js`, CSV, резервных копий и личных JSON.
5. Загрузить файлы новой папки в ветку `main`.
6. Открыть `Settings` → `Pages`.
7. В разделе `Build and deployment` выбрать `Deploy from a branch`.
8. Выбрать ветку `main`, каталог `/ (root)` и нажать `Save`.
9. После публикации открыть сайт и восстановить личную базу кнопкой `↥ Данные`.
10. Не загружать резервную копию в GitHub: она импортируется только через интерфейс уже открытого приложения.

## Перед каждым обновлением

- Использовать только релизный ZIP с пометкой «clean».
- Не копировать файлы из старого репозитория целиком.
- Проверять `git status` перед отправкой.
- Убедиться, что `.gitignore` присутствует в корне.
- Не включать облачную синхронизацию, пока не развёрнуты актуальные Firestore и Storage Rules.

## Если личный файл уже попал в коммит

Сначала сделать репозиторий приватным. Простого удаления новым коммитом недостаточно: файл останется в истории Git. Историю нужно очистить отдельно или создать новый чистый репозиторий.
