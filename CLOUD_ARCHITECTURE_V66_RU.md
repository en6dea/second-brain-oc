# Архитектура аккаунта и синхронизации V66

## Цель

Один личный аккаунт на Windows и iPhone с входом по email, Google или Apple, безопасным разрешением конфликтов и без потери локальных данных.

## Почему старую синхронизацию нельзя просто включить

Старая реализация сохраняла всё состояние одним документом Firestore. Текущая резервная копия уже близка к размеру, при котором такой документ становится ненадёжным; скриншоты сделок быстро превысят лимит. Кроме того, схема «последняя запись победила» может незаметно стереть изменения другого устройства.

## Целевая модель

- Firebase Authentication: email/password, Google и Apple.
- Firestore: отдельные документы записей внутри `users/{uid}/collections/{collection}/items/{itemId}`.
- У каждой записи: `updatedAt`, `deviceId`, `revision`, `deletedAt` и хеш содержимого.
- Скриншоты Forex: Cloud Storage в `users/{uid}/trades/{tradeId}/...`, а в Firestore — только метаданные и путь.
- Локальная база остаётся рабочей без сети; очередь изменений отправляется после восстановления соединения.
- При параллельном изменении одной записи на двух устройствах приложение показывает обе версии. Остальные изменения объединяются автоматически.
- Перед первой отправкой и перед массовым разрешением конфликтов создаётся JSON-копия.

## Безопасность

- Доступ к каждому пути разрешён только владельцу `request.auth.uid`.
- OAuth-секрет Apple хранится только на серверной стороне; в клиенте его быть не должно.
- Firebase web config не является паролем, но production-домены и правила доступа обязательны.
- Четырёхзначный PIN — локальная блокировка интерфейса, а не ключ шифрования облака.
- Для будущего сквозного шифрования понадобится отдельный recovery key; PIN для этого слишком короткий.

## Этапы внедрения

1. Подтвердить Firebase-проект и production-домен.
2. Включить email/password и Google в Firebase Authentication.
3. Подключить Apple Service ID, Team ID, Key ID и серверный private key.
4. Развернуть правила Firestore и Storage.
5. Реализовать миграцию локальных коллекций по записям и журнал изменений.
6. Проверить первый вход, новый iPhone, офлайн-правки, одновременное редактирование и восстановление доступа.
7. Только после проверки включить автоматическую синхронизацию по умолчанию.
