# Second Brain OS V83 — GameLife и цели-кампании

## Что добавлено

### GameLife
- отдельная страница персонажа;
- XP, Focus Coins и уровни развития;
- ежедневные квесты: привычка, финансы, быстрая запись и следующий шаг цели;
- дневной лимит XP, чтобы нельзя было накрутить прогресс задним числом;
- характеристики: дисциплина, финансы, знания, энергия и осознанность;
- магазин реальных наград;
- собственные награды;
- история полученного XP;
- глобальная быстрая фиксация за 30 секунд;
- салют после выполненных действий.

### Быстрая фиксация
Можно сохранить:
- мысль;
- эмоцию;
- задачу;
- идею;
- расход;
- событие.

Приложение само направляет запись в подходящий раздел. Исходные коллекции не очищаются.

### Цели
- отдельный раздел «Цели»;
- цель отображается как игровая кампания;
- мастер создания из 7 шагов;
- 3 типа целей: результат, процесс, поддержание;
- причина, метрика, срок, текущая точка;
- этапы кампании;
- обязательный следующий физический шаг на 5–20 минут;
- создание задачи из следующего шага;
- связь задач с целями и этапами;
- пауза, завершение и архивная копия при удалении;
- XP за этапы, следующие шаги и завершение цели.

## Защита данных
- перед первой загрузкой V83 создаётся автоматическая резервная копия V82.2 в IndexedDB;
- существующие записи не сбрасываются;
- старые цели автоматически дополняются новыми полями;
- обновление использует тот же ключ хранилища `secondBrainOS.v1`;
- удалённые цели сохраняются в архиве.

## Установка в GitHub

1. В текущем приложении скачайте JSON через «Экспорт данных».
2. Загрузите файлы из архива обновления в корень репозитория.
3. Обязательно замените `index.html` и `force-update.html`.
4. Если в репозитории ещё нет файлов иконок и manifest, добавьте их из полной сборки.
5. Сделайте Commit changes.
6. Дождитесь публикации GitHub Pages.
7. Один раз откройте:

`https://en6dea.github.io/second-brain-oc/force-update.html`

Страница удаляет только кэш интерфейса и не очищает личные данные.

## Файлы обновления
- `index.html` — всё приложение V83 в одном файле;
- `force-update.html` — принудительное обновление интерфейса;
- `README_V83_GAMELIFE_RU.md` — инструкция.

## Проверка
- JavaScript прошёл `node --check`;
- выполнен запуск в тестовом DOM для маршрутов `#gamelife`, `#goals`, `#today`, `#calendar`, `#finance`, `#archive`;
- проверено создание окна мастера целей;
- проверено, что миграция создаёт `state.gameLife` и не удаляет прежние массивы.
